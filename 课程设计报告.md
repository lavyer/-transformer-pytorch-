## 一、课题设计背景与意义

**机器翻译**（Machine Translation, MT）是自然语言处理领域的核心任务之一，旨在利用计算机自动将一种自然语言转换为另一种自然语言。随着全球化进程的加速和互联网信息的爆炸式增长，机器翻译在跨国交流、信息获取、跨境电商、学术研究等领域具有重要的应用价值。

传统的机器翻译方法包括基于规则的机器翻译（RBMT）和统计机器翻译（SMT），但这些方法依赖大量人工特征工程或复杂的对齐模型。**2017年，Google提出Transformer模型**，完全基于自注意力机制（Self-Attention），摒弃了循环神经网络（RNN）和卷积神经网络（CNN）的序列结构，实现了并行计算和长距离依赖的有效捕捉。Transformer已成为当前主流机器翻译系统（如Google Translate、DeepL、ChatGPT）的核心架构。

**本课程设计的目标**：
1. 深入理解Transformer的核心原理：自注意力、多头注意力、位置编码
2. 掌握PyTorch搭建Seq2Seq模型进行机器翻译的方法
3. 实现中文→英文机器翻译系统（字符级中文分词 + 词级英文分词）
4. 对比Greedy Decode和Beam Search两种解码策略的性能

## 二、设计方案概述

### 2.1 数据集介绍

选用来自Tatoeba项目的 **cmn-eng 数据集**（manythings.org/anki），包含中英双语句对，涵盖日常会话场景。

| 属性 | 数值 |
|---|---|
| 句对总数 | 29,909 对 |
| 中文平均长度 | 9.7 字符（最大44字符） |
| 英文平均长度 | 6.1 词（最大38词） |
| 中文词表大小 | 3,080（字符级） |
| 英文词表大小 | 4,853（词级，限制前10K） |
| 训练集/验证集 | 90% / 10% |
| 序列最大长度 | 50 |

**数据预处理**：
- 中文：字符级分词——在汉字间插入空格后使用WordLevel分词器
- 英文：词级分词——按空格分割后使用WordLevel分词器
- 过滤长度>200字符的异常样本

### 2.2 模型选型：Transformer

采用《Attention Is All You Need》论文的经典Transformer架构：

**编码器（Encoder）**：N层EncoderBlock堆叠
- 多头自注意力子层（Multi-Head Self-Attention）
- 前馈神经网络子层（FFN：d_model → d_ff → d_model，ReLU激活）
- 每个子层后：残差连接 + 层归一化

**解码器（Decoder）**：N层DecoderBlock堆叠
- 掩码多头自注意力子层（Masked Multi-Head Self-Attention）
- 编码器-解码器交叉注意力子层（Cross-Attention）
- 前馈神经网络子层

**位置编码**：正弦位置编码（Sinusoidal Positional Encoding）

### 3.3 Transformer 神经网络结构图

![Transformer结构图](figures/transformer_architecture.png)

**上图展示了完整的Transformer模型结构**（基于《Attention Is All You Need》论文）：
- **左侧（蓝色）为编码器（Encoder）**：输入中文Token IDs → Embedding → 位置编码 → 3层EncoderBlock（每层含多头自注意力+前馈网络+残差连接+LayerNorm）→ 最终LayerNorm
- **右侧（红色）为解码器（Decoder）**：输入英文Token IDs（右移一位）→ Embedding → 位置编码 → 3层DecoderBlock（每层含掩码自注意力+交叉注意力+前馈网络+残差连接+LayerNorm）→ 最终LayerNorm
- **交叉注意力**（红色大箭头）：解码器通过Cross-Attention从编码器获取源语言信息（K, V from Encoder, Q from Decoder）
- **输出层**：Linear Projection (d_model=256 → vocab_size=4853) → Softmax → 逐词生成英文
- **红色虚线**：残差连接（Residual Connection）
- **⊕ 符号**：逐元素相加（Element-wise Add）

### 3.3 核心代码解析

#### 3.3.1 缩放点积注意力（Scaled Dot-Product Attention）

```python
@staticmethod
def attention(query, key, value, mask, dropout):
    d_k = query.shape[-1]
    # Attention(Q,K,V) = softmax(QK^T / √d_k) V
    attention_scores = (query @ key.transpose(-2, -1)) / math.sqrt(d_k)
    if mask is not None:
        attention_scores.masked_fill_(mask == 0, -1e9)  # 将padding位置设为-∞
    attention_scores = attention_scores.softmax(dim=-1)
    if dropout is not None:
        attention_scores = dropout(attention_scores)
    return (attention_scores @ value), attention_scores
```

#### 3.3.2 多头注意力（Multi-Head Attention）

```python
def forward(self, q, k, v, mask):
    # 线性变换后分头：(batch, seq, d_model) → (batch, h, seq, d_k)
    query = self.w_q(q).view(q.shape[0], q.shape[1], self.h, self.d_k).transpose(1, 2)
    key   = self.w_k(k).view(k.shape[0], k.shape[1], self.h, self.d_k).transpose(1, 2)
    value = self.w_v(v).view(v.shape[0], v.shape[1], self.h, self.d_k).transpose(1, 2)

    # 计算注意力
    x, self.attention_scores = MultiHeadAttentionBlock.attention(
        query, key, value, mask, self.dropout)

    # 合并头：(batch, h, seq, d_k) → (batch, seq, d_model)
    x = x.transpose(1, 2).contiguous().view(x.shape[0], -1, self.h * self.d_k)
    return self.w_o(x)  # 最终线性投影
```

#### 3.3.3 Beam Search 解码（带长度惩罚）

```python
def beam_search_decode(model, source, source_mask, tokenizer_tgt,
                        max_len, device, beam_size=4, length_penalty_alpha=0.6):
    encoder_output = model.encode(source, source_mask)
    beams = [([sos_idx], 0.0, False)]  # (序列, 对数概率和, 是否完成)

    for _ in range(max_len - 1):
        all_candidates = []
        for seq, log_sum, finished in beams:
            if finished:
                all_candidates.append((seq, log_sum, finished))
                continue
            # 计算下一步token概率
            decoder_input = torch.tensor([seq], dtype=torch.long).to(device)
            decoder_mask = causal_mask(decoder_input.size(1)).type_as(source_mask).to(device)
            out = model.decode(encoder_output, source_mask, decoder_input, decoder_mask)
            log_probs = torch.log_softmax(model.project(out[:, -1]), dim=-1).squeeze(0)
            topk_log_probs, topk_idx = torch.topk(log_probs, beam_size)
            for k in range(beam_size):
                new_seq = seq + [topk_idx[k].item()]
                new_sum = log_sum + topk_log_probs[k].item()
                all_candidates.append((new_seq, new_sum, topk_idx[k].item() == eos_idx))

        # 长度惩罚分数排序
        ordered = sorted(all_candidates,
                         key=lambda c: c[1] / (len(c[0]) ** length_penalty_alpha),
                         reverse=True)
        beams = ordered[:beam_size]  # 保留Top-K
        if all(f for _, _, f in beams):
            break
    return torch.tensor(beams[0][0], dtype=torch.long, device=device)
```

#### 3.3.4 中文字符级分词

```python
def add_spaces_to_chinese(text: str) -> str:
    """在中文汉字之间添加空格，使WordLevel tokenizer可以逐字切分。"""
    text = re.sub(r'([一-鿿㐀-䶿])', r' \1 ', text)
    text = re.sub(r'\s+', ' ', text)
    return text.strip()
```

### 3.4 训练过程

**硬件环境**：Intel CPU（无GPU），16GB RAM  
**训练时间**：约2小时（15 epochs × ~8分钟/epoch）  
**总训练步数**：6,315 steps（421 batches × 15 epochs）

**学习率Warmup策略**：
- 公式：`lr = d_model^(-0.5) × min(step^(-0.5), step × warmup_steps^(-1.5))`
- warmup_steps = 4000，峰值学习率约 7.9×10^(-4)
- 学习率先线性增长至峰值，后按反平方根衰减

### 4.4 结果分析

**1. 模型效果总结**：
在约3万对中英日常会话数据上训练的8.8M参数Transformer模型，经过15个epoch训练后，BLEU达到0.152（Beam Search）。模型对**简单句**（如"他是一名医生"、"今天天气真好"）的翻译准确率很高，能够正确捕捉主谓宾结构和常用表达。

**2. Greedy vs Beam Search**：
Beam Search在所有指标上均优于Greedy Decode，尤其在复杂句子上差异明显。例如"汤姆决定不去参加玛丽的派对"：Greedy译为"Tom had to decide whether..."（语义偏差），Beam正确译为"Tom decided not to go to Mary's party"。

**3. 中文字符级分词的适用性**：
- 优势：中文词表仅3,080个字符，模型参数少，且避免了分词错误
- 劣势：丢失了词语级语义信息，字符级编码的序列更长

**4. 局限性与改进方向**：
- 训练数据规模有限（29,909对），模型对复杂长句和罕见词翻译效果不佳
- CPU训练限制了模型规模和训练轮数
- 改进方向：使用更大规模语料（WMT/OPUS）、BPE子词分词、GPU训练、预训练模型初始化
